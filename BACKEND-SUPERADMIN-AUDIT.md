# Аудит действий супер-админа

Все действия супер-админа должны попадать в общий audit log (`GET /audit`, он уже есть
и супер-админу доступен — 200). Сейчас в журнале таких событий **нет** — бан, сброс
пароля и т.п. не логируются.

## Просьба

Писать в audit log (в той же транзакции, что и само действие) при каждой
супер-операции:

| Действие | Роут | eventType | entityType | severity |
|---|---|---|---|---|
| Блокировка | `POST /users/{id}/ban` | `UserBanned` | User | warning |
| Разблокировка | `POST /users/{id}/unban` | `UserUnbanned` | User | success |
| Сброс пароля | `POST /users/{id}/password/reset` | `PasswordReset` | User | warning |
| Восстановление | `POST /users/{id}/password/restore` | `PasswordRestored` | User | success |
| Регистрация риелтора | `POST /realtors` | `RealtorRegistered` | User | success |

- `actorName` — имя/логин супер-админа, совершившего действие.
- `summary` — готовая русская строка (как у остальных событий через AuditNarrator),
  например «Заблокирован инвестор Иван Петров» / «Сброшен пароль администратора admin».
- `entityId` — id затронутого пользователя.

## Фронт

Готово: в супер-панели есть вкладка **«Аудит»**, читает `GET /audit` и показывает
время / исполнителя / описание / статус. Пока бэк не пишет супер-события — во вкладке
видны только существующие записи (тикеты, KYC, публикации). Как начнёте логировать
действия выше — они появятся в той же вкладке автоматически.
