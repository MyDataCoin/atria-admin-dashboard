# Регистрация риелтора супер-админом

Супер-админ должен создавать аккаунты риелторов. Сейчас такого эндпоинта нет — есть
только phone-OTP (инвесторы) и `POST /auth/realtor/login` (вход уже существующего).

## Просьба

### `POST /api/v1/realtors` — только SuperAdmin

Создаёт аккаунт риелтора (запись в `users` с ролью Realtor + профиль в
`realtor_profiles`).

```json
// request
{
  "username": "marat",
  "password": "temp1234",
  "fullName": "Марат Осмонов",
  "companyName": "ATRIA Realty",   // необязательно
  "phoneNumber": "+996700123456"   // необязательно
}
```

→ `201` + созданный риелтор (можно вернуть `RealtorStatsDto`-совместимый объект:
`{ id, fullName, companyName, closedDeals: 0, totalDeals: 0, blocked: false }`, чтобы
фронт сразу добавил его в список).

Ошибки:
- `400` — пустые обязательные поля (`username`/`password`/`fullName`).
- `409` — `username` уже занят.
- `403` — не SuperAdmin.

Пароль приходит от супер-админа в открытом виде (он его и задаёт) — захешируйте на
бэке как обычно. Опционально: если `password` не прислан, сгенерируйте временный и
верните его в ответе (`{ temporaryPassword }`), как в password/reset.

## Аудит

Создание риелтора — действие супер-админа, должно писаться в audit log:
`eventType: RealtorRegistered`, `entityType: User`.

## Что важно

- Роль строго SuperAdmin.
- `username` уникален (проверка → 409).
- **CORS** для прода.

## Фронт

Готово и ждёт эндпоинт:
- На вкладке «Риелторы» есть кнопка **«Зарегистрировать риелтора»** → модалка с полями
  ФИО / логин / пароль / компания / телефон.
- Форма шлёт `POST /realtors`; при успехе список риелторов обновляется, при ошибке
  причина показывается в модалке.
- Сейчас эндпоинт отдаёт **404**, поэтому регистрация вернёт ошибку; как задеплоите —
  заработает без правок фронта.
