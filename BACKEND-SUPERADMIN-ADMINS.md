# Super admin должен видеть админов (и управлять их паролями)

Супер-админ управляет паролями **админов и риелторов**. Риелторов он уже видит
(`/realtors/stats`), а **списка админов нет** — значит сбросить/восстановить пароль
админу неоткуда.

## Правила (подтверждены заказчиком)

- Сброс и восстановление пароля — **только для админов и риелторов**. У инвесторов
  пароля нет (вход по phone-OTP), поэтому этих действий для них быть не должно.
  *(На фронте уже так: у инвесторов остались только бан/разбан.)*
- Супер-админ должен **видеть админов** отдельной вкладкой.

## Чего не хватает на бэке

1. **Нет эндпоинта со списком админов.** `GET /users` — это реестр инвесторов
   (phone/KYC), админов там нет.
2. **`UserOverviewDto` не содержит `role`** — даже вернувшись в общем списке, админа
   не отличить от инвестора.

## Просьба

### `GET /api/v1/admins` — только SuperAdmin

Список staff/admin-аккаунтов, у которых есть пароль.

```json
[
  {
    "id": "uuid",           // users.id — по нему идут ban / password/*
    "fullName": "Иван Петров",
    "username": "admin",
    "email": "admin@atria.kg",
    "blocked": false
  }
]
```

- `id` — обязательно `users.id`, чтобы существующие
  `POST /users/{id}/password/reset|restore` и `/ban` работали по нему.
- Супер-админа можно включать или нет — на ваше усмотрение; если включаете, пометьте,
  чтобы он не сбросил пароль сам себе случайно (или просто не отдавайте свою строку).
- Пустой список → `[]`, не 404.

## Проверка

Под токеном superadmin:
```
GET /api/v1/admins -> 200, массив админов
```

Сброс пароля админу (уже есть на бэке, проверить что работает для admin-роли):
```
POST /api/v1/users/{adminId}/password/reset  -> 200 { temporaryPassword }
```

## Фронт

Готово и ждёт эндпоинт:
- В супер-панели добавлена вкладка **«Админы»**, читает `GET /admins`.
- Кнопки «Сбросить пароль» / «Восстановить» показываются **только** у админов и
  риелторов; у инвесторов их больше нет.
- Пока `/admins` отдаёт 404 — вкладка пустая; как задеплоите, заполнится сама.

Связанная задача (нужна тоже): открыть SuperAdmin доступ на чтение `GET /users` и
`GET /realtors/stats` — см. BACKEND-SUPERADMIN-READ-ACCESS.md, иначе и инвесторы с
риелторами не видны.
